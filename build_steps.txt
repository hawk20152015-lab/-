==========================================================
   بناء تطبيق «الصندوق» إلى APK عبر GitHub
   (كل الملفات في فولدر واحد مسطّح، بأسماء إنجليزية)
==========================================================

محتويات هذا الفولدر (كلها في الجذر، لا مجلدات فرعية، لا أسماء عربية):
  main.py                        ← كود التطبيق
  buildozer.spec                 ← إعدادات البناء
  build.yml                      ← محتوى ملف البناء (تنسخه إلى GitHub في الخطوة 3)
  README.md                      ← شرح التطبيق
  build_steps.txt                ← هذا الملف
  import_template.xlsx           ← قالب استيراد الأفراد
  icon.png                       ← أيقونة التطبيق
  IBMPlexSansArabic-Regular.ttf  ← الخط (عادي)
  IBMPlexSansArabic-SemiBold.ttf ← الخط (عريض)

----------------------------------------------------------
الخطوة 1: إنشاء مستودع جديد
----------------------------------------------------------
1. ادخل github.com وسجّل الدخول.
2. اضغط + (أعلى اليمين) ← New repository.
3. اكتب اسمًا إنجليزيًا (مثل: sandooq) ← Public ← Create repository.

----------------------------------------------------------
الخطوة 2: رفع الملفات إلى جذر المستودع
----------------------------------------------------------
4. Add file ← Upload files.
5. اسحب كل هذه الملفات من الفولدر إلى الصفحة:
       main.py
       buildozer.spec
       README.md
       import_template.xlsx
       icon.png
       IBMPlexSansArabic-Regular.ttf
       IBMPlexSansArabic-SemiBold.ttf
   (build.yml وbuild_steps.txt اختياريان — لن يضرّا إن رفعتهما.)
6. Commit changes.

   تأكد في تبويب Code أن الملفات في الجذر مباشرة (وليست داخل مجلد).

----------------------------------------------------------
الخطوة 3: إنشاء ملف البناء على GitHub مباشرة
----------------------------------------------------------
(بهذه الطريقة لا تتعامل مع أي مجلد مخفي على حاسوبك)

7. افتح تبويب Actions في مستودعك.
8. اضغط الرابط "set up a workflow yourself".
9. احذف كل المحتوى في المحرّر.
10. افتح ملف build.yml بأي محرّر نصوص، انسخ كل محتواه، والصقه في المحرّر.
11. اضغط Commit changes ← Commit.

   (سيُنشئ GitHub المجلد .github/workflows تلقائيًا.)

----------------------------------------------------------
الخطوة 4: تشغيل البناء وتنزيل الـ APK
----------------------------------------------------------
12. سيبدأ البناء تلقائيًا (أول مرة: 20–40 دقيقة).
    يدويًا: Actions ← Build Android APK ← Run workflow.
13. عند ظهور العلامة الخضراء، انزل إلى Artifacts في صفحة التشغيل.
14. نزّل hesabat-apk (ملف zip) ← فك ضغطه ← بداخله ملف .apk.
15. انقله للهاتف وثبّته (فعّل «تثبيت من مصادر غير معروفة» إن طُلب).

----------------------------------------------------------
إذا فشل البناء
----------------------------------------------------------
افتح التشغيل الفاشل ← الخطوة الحمراء «Build APK with Buildozer»
← انسخ آخر 20-30 سطرًا وأرسلها للمطوّر.
